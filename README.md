# QuickTables 🧮

A fast-paced multiplication tables trainer, built as a tribute to a Windows 95/98-era
educational game from primary school that never made it past a single school's computers.
The original's author no longer wants to maintain or share it, so this is a from-scratch
reimplementation of the concept — same core idea, modern shell.

**[Play it here](https://florisco.github.io/QuickTables)**

## How it works

You get a grid of times tables (up to 12×12 by default). Each round the game picks a
random unsolved cell and asks for the product — type the answer (or use the on-screen
keypad) and confirm. Correct answers are worth a point and that cell is solved for good;
wrong or too-slow answers just move on to a new cell. See how many you can clear before
the timer runs out.

- **Configurable timing** — total game length and time-per-question both adjustable
- **Configurable grid size** — tables up to any size from 2×2 to 12×12
- **Light & dark themes**, following your system preference by default
- **Dutch, English, and French**, auto-detected from your browser language
- **On-screen keypad**, for touch devices or anyone who'd rather click than type
- **A printable PDF certificate** at the end of a game, with your name, score, and settings
- Optional **local high-score and settings memory** — nothing ever leaves your device;
  see [Privacy](#privacy--storage) below

## Play online

Open the hosted page (link above) on any modern browser — desktop, tablet, or phone.
Nothing to install.

## Play offline

The entire game is a single, self-contained HTML file with no external dependencies,
no CDN calls, no build step, and no server. That means you can:

1. Download [`index.html`](./index.html) from this repository (or clone the repo).
2. Double-click it, or open it in any browser via `File → Open`.
3. Play exactly as you would online — the only thing you lose offline is the initial
   language auto-detection convenience and browser-storage persistence in some very
   locked-down setups (see below); everything else works identically.

This makes it easy to hand out on a USB stick, put on a school's shared drive, or keep
around long after any particular website or hosting arrangement is gone — which is
exactly the kind of quiet disappearance that made the original game unrecoverable in the
first place.

## Privacy & storage

If you choose to, the game can remember your name, settings, language, theme, and
personal-best scores on your own device between visits, using `localStorage`. You're
asked once, with a simple **Save / Don't save** choice — nothing is ever sent to a
server, because there is no server. You can clear this data at any time from the
settings screen. If storage isn't available (private browsing, some offline/`file://`
setups), the game simply runs from memory for that session, no worse than before.

I'm sharing this openly because I think it's a fair and interesting way to build a small
nostalgia project, not something to hide. If that changes how much you trust or want to
build on this code, that's a completely reasonable position — please do read through it
yourself, especially before deploying it anywhere with real students.
